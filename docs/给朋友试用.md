# 小灵访谈 · 给朋友远程试用（小白分步说明）

本文说明：**你（主人）**要在自己电脑上准备什么、怎么操作；**朋友**那边只要做什么；以及常见问题和安全注意。不要求懂编程，按顺序做即可。

---

## 一、先搞清两件事

| 角色 | 要做什么 |
|------|----------|
| **你（主人）** | 电脑里装好项目、配好密钥、**开着电脑**并运行程序，再用「隧道」把网站临时暴露到公网，把链接发给朋友。 |
| **朋友** | 用手机或电脑浏览器**打开你发的链接**，用你给的**普通账号**登录、使用访谈功能。 |

**重要：** 朋友试用时，你的电脑要尽量一直开机、不要休眠；两个终端窗口（见下文）要一直开着。你关掉程序或断网，朋友就打不开。

---

## 二、你（主人）需要准备什么

### 2.1 硬件与网络

- 一台 **Mac 或 Windows 电脑**（你平时开发用的那台即可）。
- **能上网**（家里 Wi‑Fi 即可，不必和朋友同一局域网）。

### 2.2 要安装的软件（只做一次）

1. **Node.js**（建议 20 或以上）  
   - 官网下载安装：<https://nodejs.org/>  
   - 装好后打开「终端」（Mac）或「命令提示符 / PowerShell」（Windows），输入下面命令，若能看到版本号就成功了：
     ```bash
     node -v
     npm -v
     ```

2. **Cloudflare Tunnel 的命令行工具 `cloudflared`**（用来生成给朋友用的 https 链接）  
   - **Mac**（已安装 [Homebrew](https://brew.sh/) 时）在终端执行：
     ```bash
     brew install cloudflare/cloudflare/cloudflared
     ```
     若提示没有 brew，可先按 Homebrew 官网安装，或到 Cloudflare 文档页下载安装包：  
     <https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/>  
   - **Windows**：到上面同一页面下载 Windows 版安装包，按提示安装；安装完成后在命令行输入 `cloudflared --version` 能显示版本即可。

3. **代码项目**  
   - 你已经把本仓库（AI 面试官 / 小灵访谈）放在电脑里，并能进入项目文件夹（里面有 `package.json`）。

### 2.3 账号与密钥（必须配置）

项目依赖 **MiniMax** 的 API 才能对话，你需要有密钥并在配置文件里写好。

在项目根目录有一个 **`.env`** 文件（若没有，把 `.env.example` 复制一份改名为 `.env`）。至少需要关心这些项：

| 配置项 | 说明 |
|--------|------|
| `MINIMAX_API_KEY` | MiniMax 的 API Key，按你账号后台说明填写。 |
| `ADMIN_USERNAME` | 管理员登录名，默认可以是 `admin`。 |
| `ADMIN_PASSWORD` | **管理员密码**，请设成**强密码**，不要告诉别人。 |
| `SESSION_PASSWORD` | **只要用隧道给朋友公网访问，就必须设置**，至少 **32 位**随机字符。可在终端生成一行：Mac/Linux：`openssl rand -base64 32`；Windows 可用在线密码生成器生成一长串。 |

**不要**把 `.env` 文件或截图发到公开群聊；**不要**把管理员密码发给朋友。

## 三、你（主人）第一次使用前要做什么（只做一次）

在终端里进入**项目根目录**（能看到 `package.json` 的那一层），依次执行：

```bash
npm install
```

```bash
npx prisma migrate dev
```

（若还没有 `.env`，先：`cp .env.example .env`，再编辑 `.env` 填好上面的密钥和密码。）

确认本机能打开网站：

```bash
npm run dev
```

浏览器访问：**http://localhost:3500**  
能打开登录页就说明基本环境 OK。按 `Ctrl + C` 可先停掉这次运行。

**创建管理员（若还没登进去过）：**  
在 `.env` 里已写好 `ADMIN_USERNAME`、`ADMIN_PASSWORD` 后，再执行 `npm run dev`，浏览器打开 http://localhost:3500 ，用这组账号登录；系统会自动创建管理员。登录后进 **`/admin`** 后台。

**给朋友用的账号：**  
在 **`/admin`** 里给 Ta 新建一个 **普通用户**（不要给管理员权限），记下**用户名和密码**，通过微信等方式单独发给朋友（不要发在公开群）。

## 四、你（主人）每次给朋友试用时要怎么做

下面每次试用都做一遍，**顺序不要反**。

### 步骤 1：打开第一个终端 —— 启动网站

进入项目根目录，执行：

```bash
npm run dev
```

保持这个窗口**不要关**。看到类似 “Ready” 且提示端口 **3500** 即可。

### 步骤 2：打开第二个终端 —— 启动隧道

**再新开一个**终端窗口，同样进入项目根目录，执行：

```bash
npm run tunnel
```

稍等几秒，里面会出现一行 **以 `https://` 开头、以 `trycloudflare.com` 结尾** 的地址，例如：

`https://xxxx-xxxx.trycloudflare.com`

**这一整段就是发给朋友的链接。**  
复制下来，通过微信 / QQ 等私聊发给朋友。

这个窗口也**不要关**；关掉隧道，链接立刻失效。

### 步骤 3：把链接和账号发给朋友

私聊发给朋友两样东西：

1. 上面复制的 **https 完整链接**（每次重启 `npm run tunnel` 链接可能会变，以**最新一次**终端里显示的为准）。  
2. 你在 `/admin` 里给 Ta 建的 **普通用户名和密码**。

**不要发：** 管理员密码、`.env`、MiniMax Key。

### 步骤 4：试用结束后（可选）

- 朋友用完你可以在两个终端里按 `Ctrl + C` 停止；  
- 或最小化窗口保持运行，方便 Ta 继续试。

## 五、朋友需要准备什么、怎么做

### 5.1 朋友需要什么

- 一部 **智能手机** 或 **电脑**。  
- 能上网（4G/5G/Wi‑Fi 均可）。  
- 任意常见浏览器：**Safari、Chrome、微信内置浏览器**等一般都可以。  
- **不需要**安装 Node、不需要安装 cloudflared。  
- **不需要**和你连同一 Wi‑Fi。

### 5.2 朋友操作步骤

1. 打开你私聊发来的 **https 链接**（整段复制到浏览器地址栏打开，不要漏字）。  
2. 若出现「打不开」「502」：让你那边确认 **两个终端是否都在运行**、你是否用的是 **3500** 端口且先开了 `npm run dev` 再开隧道。  
3. 在登录页输入你给的 **用户名、密码**（注意大小写若你创建时有区分）。  
4. 登录成功后按页面提示使用「访谈」等功能即可。  
5. **不要**把登录链接转发给陌生人；这是临时试用链接，且相当于访问你的电脑上的服务。

## 六、常见问题（小白向）

**1. 朋友打开链接显示 502**  
- 你这边要先运行 `npm run dev`，再运行 `npm run tunnel`。  
- 本机浏览器打开 http://localhost:3500 是否正常？本机不正常就先解决本机。

**2. 朋友能打开登录页，但一登录又回到登录页**  
- 确认 `.env` 里已设置 **`SESSION_PASSWORD`**（至少 32 位），保存后**重启** `npm run dev`。  
- 让朋友**完全关闭浏览器标签**再重新打开你发的 **最新** https 链接；必要时用手机浏览器「清除该网站数据」后再试。

**3. 提示用户名或密码错误**  
- 核对 `/admin` 里给 Ta 建的用户名、密码；注意是否有多余空格。  
- 不要用管理员账号给朋友试（除非你愿意）；建议单独建普通用户。

**4. 每次隧道链接都不一样**  
- 临时隧道（Quick Tunnel）就是这样；**每次**执行 `npm run tunnel` 后，以**终端里最新打印的链接**为准发给朋友。

**5. Windows 没有 openssl 怎么生成 SESSION_PASSWORD**  
- 用浏览器搜索「随机密码生成器」，生成一段 **至少 32 位** 的字母数字组合，粘贴进 `.env` 的 `SESSION_PASSWORD=` 后面。

**6. cloudflared 安装失败**  
- 到官方下载页选择对应系统安装包手动安装：  
  <https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/>

---

## 七、安全提醒（很重要）

- 隧道等于把你的网站临时暴露在公网上，**务必**设置强 `ADMIN_PASSWORD` 和 **`SESSION_PASSWORD`**。  
- 只把 **https 链接** 和 **普通用户密码** 发给信任的人。  
- 试用结束可关掉两个终端，链接即失效。  
- 正式长期对外服务请用服务器部署 + HTTPS，见项目根目录 `README.md` 里的「外网部署关键点」。
